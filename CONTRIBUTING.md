# Contributing Guide

Thank you for contributing to Amaanitvam Platform.

## Development Workflow

1. Create an issue.
2. Create a feature branch.
3. Implement changes.
4. Submit a pull request.
5. Obtain review approval before merging.

## Branch Naming

```text
feature/feature-name
bugfix/bug-name
docs/document-name
```

## Commit Convention

```text
feat: add certificate verification page
fix: resolve authentication bug
docs: update onboarding guide
refactor: improve api structure
```

## Pull Requests

All changes must be submitted through pull requests.

Direct commits to the main branch are not permitted.

## Code Standards

* Follow project structure.
* Write clear and maintainable code.
* Update documentation when necessary.
* Keep pull requests focused and small.

## Security

Do not commit:

* Secrets
* API keys
* Database credentials
* Environment files

Report security concerns to the technical team.
